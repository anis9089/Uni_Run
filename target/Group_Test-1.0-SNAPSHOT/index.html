01. Introduction

Redefining Campus Athletics.

Uni-Run is a cutting-edge Marathon & Virtual Run Coordinator. We bridge the gap between physical track events and remote fitness tracking, providing a unified platform for students to compete, track, and celebrate fitness milestones in a session-secured environment built on Java MVC.

02. Core Objectives

/01

Seamless event lifecycle management from sign-up to bib assignment.

/02

Real-time community telemetry via high-impact data visualization.

/03

Robust security using server-side session management.

/04

Centralized result logging for administrative transparency.

Project Scope

Runners

Profile management, event discovery, virtual time submission.

Administrators

Full CRUD on events, bib batching, and verified result approval.

Database Entity Relationship Diagram (ERD)

Following the requirement of at least 4 tables.

Users (Account)
PK userID (INT)
username (VARCHAR)
password (VARCHAR)
role (ENUM: 'ADMIN', 'RUNNER')
Participants
PK participantID (INT)
FK userID (INT)
fullName (VARCHAR)
gender (CHAR)
age (INT)
email (VARCHAR)
Events
PK eventID (INT)
eventName (VARCHAR)
date (DATE)
distanceKM (DOUBLE)
location (VARCHAR)
Registrations
PK regID (INT)
FK participantID (INT)
FK eventID (INT)
bibNumber (VARCHAR)
finishTime (TIME)
status (VARCHAR)
Relationships:
  • One User has one Participant profile (1:1).
  • One Participant can have many Registrations (1:N).
  • One Event can have many Registrations (1:N).

Java MVC System Flow

Login (JSP): User enters credentials. Validated by AuthServlet. Session created.
↓
Dashboard (JSP): Fetches summary data via DashboardServlet using EventBean.
↓
Event Management (CRUD):

Admin uses ManageEventServlet to update finisher times. Participant uses RegisterServlet to join runs.
